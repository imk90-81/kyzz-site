## 1. VS Code 供应链攻击席卷 GitHub / OpenAI / Mistral AI，3,800 内部仓库泄露

**事件概述**

代号 **Mini Shai-Hulud** 的供应链攻击由威胁组织 TeamPCP 发起，通过污染上游 npm 包（TanStack 生态，170+ 包）→ 劫持 Nx Console VS Code 扩展 → 窃取开发者凭证的三级链路，在 **仅 18 分钟** 窗口期内成功入侵：

| 受害方 | 影响范围 |
|--------|---------|
| GitHub | ~3,800 个内部代码仓库被窃取，凭证穿越 CI/CD 管道 |
| OpenAI | 2 台员工设备沦陷，部分内部源码和凭证外泄 |
| Mistral AI | SDK 被植入木马，代码仓库已在暗网公开出售 |

**关注理由** ⚠️

> 这不是一次普通的 CVE 漏洞，而是对 **AI 编程工具链** 的系统性攻击。攻击者特意将恶意命令伪装为 MCP（Model Context Protocol）设置任务，Anthropic Claude 配置文件被列为优先窃取目标。对于每天使用 Claude Code / Cursor / Copilot 的开发者而言，**你的 AI 编码助手配置本身已成为高价值攻击面**。建议立即检查 VS Code 扩展版本，审计 npm 依赖树，并关注 GitHub / OpenAI 的后续披露。

## 4. Gemini 3.5 Flash 稳定版上线：编程与 Agent 任务基准全面超越 Claude Opus 4.7 / GPT-5.5

**事件概述**

Google 发布 Gemini 3.5 Flash 稳定版（`gemini-3.5-flash`）：
- 在 **5 项编程与 Agentic 评测** 中领先所有竞品，包括 Claude Opus 4.7 和 GPT-5.5
- 推理速度：公开 API **4 倍加速**，Antigravity 2.0 内部优化达 **12 倍**
- 定价：$1.50 / $9（每百万 tokens 输入/输出），显著低于 Claude Sonnet 4.6（$3 / $15）
- 弱项：长上下文（128k+）和知识广度评测仍有差距

**关注理由** 🚀

> Google 的策略非常清晰：**用低价高速模型抢占开发者心智**。$1.50 的输入价格仅为 Claude 的 1/2，速度却快 4 倍。结合 Antigravity 2.0 平台，Google 正在构建一个 **"足够好 + 极致性价比"** 的编程 Agent 生态。对于使用 Cursor / Claude Code 的开发者，建议将 Gemini 3.5 Flash 纳入备选模型池——尤其在需要高频 API 调用、成本敏感的场景下，它可能成为最优解。

## 5. 中国 AI 模型占 OpenRouter 60% 份额，"顾问模型"架构兴起重塑开发者选型

**事件概述**

Databricks CEO Ali Ghodsi 披露关键数据：
- 中国模型在 OpenRouter 的调用量从 2024 年约 **1%** 飙升至 2026 年 5 月 **60%+**
- 主力模型：DeepSeek V3.2（$0.28/$0.42）、Kimi K2.6（GPQA 90.5）、智谱 GLM-5.1
- 10 项标准评测总成本对比：Claude $4,811 vs DeepSeek $1,071 vs 智谱 GLM $544

企业正采用 **"顾问模型"架构**：日常任务使用低价中国模型，仅在高价值场景调用 OpenAI / Anthropic。

**关注理由** 🔄

> 这是 2026 年 AI 编程生态最被低估的结构性变化。**60% 的市场份额意味着中国模型已经不再是"平替"，而是主流选择。** 对于 AI 编程工具的开发者和用户，这直接影响：
> 1. **工具选型**：Cursor Composer 2.5 用 Kimi K2.5 基座、Claude Code 接入多模型支持，说明头部工具已押注多模型架构
> 2. **成本结构**：企业 AI 编程预算可能从"按年采购 Claude/Copilot"转向"混合编排多模型"
> 3. **职业发展**：掌握 DeepSeek/Kimi/GLM 的工程师将成为 2027 年企业 AI 基础设施建设的核心决策者
