## Google I/O 2026 明日开幕

Google I/O 2026 将于美西时间5月19日上午10点举行。Google 已确认 keynote 将覆盖"最新 Gemini 模型更新"和 **"Agentic Coding（智能体编程）"**——被广泛解读为 Gemini 4.0 的正式发布。

**预期发布：**

- **Gemini 4.0** — 旗舰模型升级，多模态推理和 Agent 可靠性大幅提升
- **Google Cloud Agentic Toolkit** — 企业级 Agent 部署的 API 扩展与定价细节
- **Aluminium OS** — 基于 Android 的 ChromeOS 替代品，预计 2026 年发布

> 这是 2026 年 AI 编程领域最重要的单一事件。如果 Gemini 4.0 能在 agentic coding 场景实现突破，将直接改变 Copilot/Cursor/Windsurf 等工具的底层模型竞争格局。

## 苹果评估 AI 编程应用政策

据 The Information 5月13日报道，苹果正在内部讨论是否允许 **Agentic Coding / Vibe Coding 类应用**入驻 App Store。

**两大顾虑：**

1. **安全审核失效风险** — AI 生成的代码无法预先审查
2. **收入侵蚀** — 这类应用可能削弱 App Store 分发和抽成收入

> 这是一个具有行业标杆意义的政策信号。如果苹果最终开放，将标志着主流移动平台正式接纳 AI Agent 编程范式。

## Skar：Agent 回归测试工具

开源工具 **Skar** 于 5月15日发布，核心功能是将捕获的 AI coding agent 运行轨迹自动转换为 **pytest 回归测试**。

**解决的痛点：** AI 编码 Agent 的行为可靠性难以保障——同样的 prompt 可能产生不同结果。Skar 让开发者把"成功运行"固化为可复用的测试用例。

> 随着 AI 编程 agent 投入生产环境，如何测试和保证 agent 行为的正确性成为紧迫课题。Skar 代表了"agent 行为回归测试"的重要方向。
