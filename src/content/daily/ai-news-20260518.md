### 4️⃣ 苹果评估 AI 编程类应用入驻 App Store 方案

**事件：** 据 The Information 5月13日报道，苹果正在内部讨论是否允许 **Agentic Coding / Vibe Coding 类应用**入驻 App Store。App Store 现有审核规则明确禁止在 iPhone/iPad 上直接编程并生成其他应用。

**苹果的两大顾虑：**
1. **安全审核失效风险**：AI 生成的代码无法预先审查，可能绕过安全检查引入恶意软件
2. **收入侵蚀**：这类应用可能减少用户购买其他 App，削弱 App Store 分发和抽成收入

目前苹果正在探索折中路线——设计一套让 AI 编程应用遵守隐私和安全标准的系统。

**📌 值得关注的原因：**
这是一个具有**行业标杆意义**的政策信号。如果苹果最终开放（哪怕附带严格限制），将标志着主流移动平台正式接纳 AI Agent 编程范式。反之，若持续封锁，AI 编程工具将被限制在桌面/Web 端，移动端 Agent 编程生态的发展将严重滞后。对所有做 AI 编程工具的公司来说，这个决策直接影响产品路线图。

### 5️⃣ Skar：将 AI Agent 编码轨迹转化为 Pytest 回归测试（开源新工具）

**事件：** 开源工具 **Skar** 于 5月15日发布，核心功能是将捕获的 AI coding agent 运行轨迹自动转换为 **pytest 回归测试**。

**解决的核心痛点：** AI 编码 Agent 的行为可靠性难以保障——同样的 prompt 可能产生不同结果，回归问题难以追踪。Skar 让开发者可以把 agent 的"成功运行"固化为可复用的测试用例。

**社区反响：** Hacker News 350 分（254 条评论），热度不俗。

**📌 值得关注的原因：**
随着 AI 编程 agent 越来越多地投入生产环境，**如何测试和保证 agent 行为的正确性**成为紧迫课题。Skar 代表了一个重要方向：不是试图让 agent 完美无误，而是建立"agent 行为回归测试"的基础设施。这与 PwC 用 Claude Code 交付生产软件的趋势高度呼应——当 AI 编码从玩具走向生产，工程质量保障工具就会成为刚需。
