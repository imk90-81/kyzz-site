研究警示

LLM Agent 在后端代码生成中存在"约束衰减"：学术研究揭示结构化编码的系统性弱点

arXiv 5月7日提交的一项系统研究揭示，大模型 Agent 在处理多文件后端代码生成时出现 "约束衰减"（Constraint Attenuation） 现象——随着结构性要求的累积，Agent 逐步"遗忘"早期约束。

维度关键数据
测试规模 80 个新项目 + 20 个功能实现，覆盖 8 种 Web 框架
性能下降 部分配置 assertion 通过率平均下降 30 个百分点
框架差异 宽松框架（Flask）尚可；强约定框架（FastAPI / Django）大幅下滑
根本原因 结构性要求累积 → Agent 逐步"遗忘"早期约束
关注理由

这是 2026 年对 AI 编程 Agent 工程可靠性 最硬核的实证警告。当前 Vibe Coding 浪潮中，多数展示聚焦于"从零生成一个简单 App"——但真实工程场景要求 AI 在强约定、多配置、跨文件的复杂约束下持续保持正确性。在企业级项目中，AI 编程 Agent 的输出仍需系统化人工审查，尤其是在 FastAPI / Django / Spring 等强约定框架下。这也解释了为何 Claude Code 的 /code-review 命令在此刻上线——代码审查不是锦上添花，而是弥补 Agent 可靠性短板的必要环节。
